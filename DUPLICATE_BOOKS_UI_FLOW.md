## Use Case Examples

### Example 1: Casual Reader
**Situation**: User accidentally tries to add a book they already have.
**Action**: Click "View Existing Entry" to see they already read it.
**Result**: No duplicate created, user reminded of existing book.

### Example 2: Library Manager
**Situation**: Library receives a second copy of a popular book.
**Action**: Click "Increment Count (+1)" to track total inventory.
**Result**: Quantity updated to 2, easy to see they have multiple copies.

### Example 3: Book Collector
**Situation**: Collector buys a first edition to go with their paperback.
**Action**: Click "Add as Separate Entry" to track each edition separately.
**Result**: Two distinct entries, each with unique photos and notes about condition, edition details, etc.

## Technical Notes

### Button States
- **Default**: Blue outline (increment), green outline (separate), gray outline (navigate)
- **Hover**: Solid color background
- **Loading**: Spinner icon + "Updating..." / "Creating..." / "Navigating..."
- **Disabled**: Grayed out during API call

### Animation
- Modal fades in smoothly (Bootstrap default)
- Button state changes are instant for responsiveness
- Redirect happens immediately after successful API response

### Accessibility
- All buttons have clear labels and icons
- Keyboard navigation supported (Tab through options)
- ARIA labels for screen readers
- Modal can be dismissed with Escape key (unless backdrop is static)
- Focus trapped within modal while open

## Summary

The duplicate books feature provides a clean, intuitive interface for handling a common scenario. Users get clear choices with obvious outcomes, making the library management experience more flexible and powerful while maintaining the safety of duplicate detection.
